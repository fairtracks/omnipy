# ELIXIR All-Hands 2025 Abstract

## Redefining data interoperability with Omnipy
_The time is ripe to challenge fundamental assumptions on how to wrangle
and interconnect your data!_

Data models are key to interoperability, but have issues. They are often
implemented too loosely, as free-for-all spreadsheets inviting chaos, or
too rigidly, with validation that does not allow a single mismatch!
Applying the models to real-world data often means meticulous and mundane
work.

The slogan "Parse, Don't Validate" is making rounds in programming circles
– but the concept is not new. A "parser" allows large variation in inputs
while producing structured outputs. Dynamic programming languages like
Python are great at parsing data, but lack of strict typing means bad data
can wreak havoc at any time. Now, cutting-edge support for explicit types
in Python allows the best of both worlds – strict and dynamic typing –
provided through Omnipy.

Omnipy is a new Python library that offers a systematic and scalable
approach to research data and metadata wrangling. It provides interactive,
hands-on access to local data or outputs from remote APIs – all parsed
into data models that are as general or specific as you want.  You can
import, browse and convert any data. It is just as simple to work with
single files as with batch datasets. When you’re happy with a dataflow,
you can seamlessly scale it up for remote deployment on compute
infrastructures.

Developed in ELIXIR Norway, Omnipy is perfect for shaping training data
for AI. We aim to develop Omnipy into a community effort by building and
encouraging integrations.
