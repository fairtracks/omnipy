# Omnipy — Pydantic v2 native migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Migrate Omnipy to **native Pydantic v2 APIs** while preserving Omnipy’s **public behavior**.

**Architecture:** Keep `omnipy.util.pydantic` as the single internal Pydantic import facade, but make it **v2-only** and add **small, Omnipy-specific v2 helpers** where Omnipy previously depended on v1 internals.

**Tech Stack:** Python 3.10+, Pydantic v2, Prefect 2.x (<3), uv, pytest (+ pytest-mypy-plugins).

- **Recommended migration aid from the guide:** `bump-pydantic` (beta)

## Non-goals (per spec)

- No `pydantic.v1` usage (including “temporary” compatibility).
- No Prefect 3 migration.
- No broad refactor / TODO sweep beyond code that must change.

## Current hotspots (from code scan + spec)

High churn is expected around:

- `src/omnipy/util/pydantic.py` (currently imports `pydantic.v1` and re-exports v1 APIs)
- v1 internals usage:
  - `__fields__`, `ModelField`, `validate_model`, `ErrorWrapper`, manual `ValidationError(...)`
- v1 decorators:
  - `@validator`, `@root_validator`

Representative call sites:

- `src/omnipy/data/model.py`, `src/omnipy/data/dataset.py`, `src/omnipy/data/param.py`
- `src/omnipy/util/publisher.py`
- `src/omnipy/components/tables/models.py`, `src/omnipy/components/remote/models.py`
- `src/omnipy/data/_data_class_creator.py`, `src/omnipy/data/_display/integrations/jupyter/helpers.py`
